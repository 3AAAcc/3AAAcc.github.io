VS Code 中的 Git 基础操作指南

2024-05-20 学习笔记 / Git教程 Wenjun Zhu

在使用 VS Code 进行项目开发时，掌握基础的 Git 操作能够极大地提升我们的代码管理效率，尤其是在团队协作和版本控制方面。本文根据我的学习笔记，总结了在 VS Code 中最常用的几个 Git 图形化操作步骤。

1. 将本地的文件夹链接到远程仓库

当我们在本地初始化了一个新的项目文件后，第一步通常是将其推送到 GitHub 等云端仓库进行托管：

  • 在 VS Code 的“源代码管理（Source Control）”面板中，点击 “发布分支 (Publish Branch)” 按钮。
  • 此时，VS Code 的顶部会弹出一个文本框，提示你 输入你要创建的仓库名字（默认就是你本地项目文件夹的名字）。
  • 确认名称并选择发布到 Public（公开）或 Private（私有）仓库后，点击推送。
提示：完成以上操作后，Git 会自动将所有本地的代码提交（Commit）并更新推送到 GitHub 上，你的远程仓库就建立成功了。

2. .gitignore 文件的使用

在开发过程中，往往会有一些编译生成的中间文件、环境配置文件或缓存（例如 Python 的 __pycache__），这些文件是不需要、也不能推送到远端仓库的。

在某文件或文件夹 还没有被提交（Commit）之前，我们可以在项目根目录下创建或编辑 .gitignore 文件，在其中输入该文件的相对路径或匹配规则。保存后，Git 的源代码管理面板就会自动忽略这些文件，防止误传。

3. 合作时本地代码的拉取与同步

在团队协作中，云端的代码可能会被其他人修改。为了保证代码的一致性并防止冲突，我们需要掌握拉取与同步的区别：

操作 1：获取 (Fetch)

点击源代码管理面板顶部的刷新/获取按钮（通常是一个带向下箭头的云朵图标），这会将所有云端的最新提交记录拉取下来，但并不会更改你本地的实际文件内容。这个操作仅仅是为了让你“知道”项目在云端上被其他人更改了什么。

执行 Fetch 操作后，你可能会看到云端的版本已经领先于你的本地版本。

操作 2：同步更改 (Sync Changes / Pull)

当你确认了云端的更改并且准备好更新本地代码时，点击 “同步更改 (Sync Changes)” 按钮（图标通常是相互追逐的上下箭头）。这个操作会将本地的内容正式同步更新为刚才拉取下来的最新提交版本。

注意：在点击“同步”前，最好确保你本地当前的修改已经 Commit 暂存，以免在合并云端代码时造成本地未保存进度的丢失。
